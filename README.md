# Forkline

中文 Web 端 Git 可视化管理工具。无需安装 npm 依赖，使用 Node.js 内置 HTTP 服务启动。

## 功能

- GitKraken 风格提交图谱，全部分支模式下以 `main` / `master` 为主线。
- 本地分支、远端分支、工作区状态和未暂存更改查看；本地分支会显示 upstream、领先/落后数量和上游丢失状态。
- 文件路径树状归纳，支持同层级聚合与滚动查看。
- 工作区 Diff、历史提交 Diff、小窗口预览和最大化对照。
- 可拖拽左侧栏、右侧栏、底部变更面板、文件列表宽度和提交框高度。
- 支持抓取、拉取、智能推送，并在完成后显示中文同步摘要：当前 upstream、领先/落后变化、远端新增/更新/删除引用；没有 upstream 的当前分支会自动使用 `git push -u` 设置跟踪关系。
- 支持暂存全部、单文件与 Ctrl / Shift 多选暂存 / 取消暂存、丢弃未暂存或已暂存改动、创建提交和追加提交。
- 支持创建储藏、储藏所选文件、查看储藏 Diff、应用 / 弹出 / 删除储藏。
- 支持本地分支切换、远端分支签出为本地跟踪分支、远端分支右键删除、保留 merge commit 的分支合并、新建/重命名/删除分支。
- 支持创建 Tag、标签列表、复制/查看 Tag、删除本地 Tag、推送 Tag、删除远端 Tag，相关动作也集成在 Tag 右键菜单中。
- 支持合并冲突工作流：冲突时工作区会显示继续 / 中止合并。
- 支持从分支右键菜单把当前分支变基到目标分支；变基冲突时工作区会显示继续 / 跳过 / 中止变基。
- 支持在提交右键菜单和提交详情中挑选提交到当前分支；merge 提交会先选择主线，冲突时工作区会显示继续 / 跳过 / 中止挑选。
- 支持在提交右键菜单和提交详情中还原提交；merge 提交会先选择主线；支持软 / 混合 / 硬重置到指定提交，并在按钮旁标注对应 Git 指令。
- 支持修改提交信息：最新提交使用 amend，历史提交使用 rebase reword；提交详情和右键菜单支持把非 merge 提交压缩进父提交、修补进父提交或从历史中丢弃。

## 启动

```bat
start.cmd
```

打开：

```text
http://127.0.0.1:5177
```

默认显示示例数据。顶部输入本机 Git 仓库路径后点击 `打开`，即可读取真实仓库。

## 注意

真实仓库模式下的写入操作会先弹窗确认，再由本地 Node 服务执行 Git 命令。修改历史提交信息和变基都会重写相关 SHA，工作区不干净时会被阻止。

抓取、拉取和推送完成后，Forkline 会显示同步摘要。摘要会说明当前分支对应的 upstream、本地领先/落后数量如何变化，以及这次远端引用新增、更新或删除了什么。

推送当前分支时，如果已经设置 upstream，Forkline 会执行普通 `git push`；如果没有 upstream，会选择 `origin` 或第一个可用远端执行 `git push -u <远端> <当前分支>`。删除远端分支会执行 `git push <远端> --delete <分支>`，只删除远端仓库中的分支，不会删除本地分支。

推送 Tag 会执行 `git push <远端> refs/tags/<tag>:refs/tags/<tag>`；删除远端 Tag 会执行 `git push <远端> :refs/tags/<tag>`，不会删除本地 Tag。本地删除 Tag 使用 `git tag -d <tag>`，不会影响远端 Tag。

对 merge 提交执行挑选或还原时，Forkline 会要求选择主线，对应 Git 的 `-m <父提交编号>`。通常父提交 1 是执行合并时所在的分支方向，父提交 2 是被合并进来的分支方向。

如果 Git 提示 `index.lock`，Forkline 会在中文提示里显示刚才执行的操作、锁文件路径/时间，以及能检测到的 Git 进程。`index.lock` 本身不记录具体命令；确认没有 Git 或编辑器正在操作仓库后，才能删除这个锁文件重试。

合并分支时如果出现冲突，工作区会标出冲突文件，并显示“继续合并 / 中止合并”。解决冲突并暂存后继续；不想保留这次合并则中止。

变基当前分支到其他分支时，Forkline 会执行 `git rebase <目标分支>`。这会重写当前分支中被重放提交的 SHA；如果这些提交已经推给别人协作，应先确认团队是否允许。变基冲突时，工作区会标出冲突文件，并显示“继续变基 / 跳过变基 / 中止变基”。解决冲突并暂存后继续；不想保留当前冲突提交可跳过；想回到变基前状态则中止。

历史编辑里的“压缩进父提交 / 修补进父提交 / 丢弃此提交”对应 `git rebase -i` 的 `squash`、`fixup`、`drop`。这些操作会重写当前分支历史 SHA；Forkline 会要求工作区干净，并暂时拒绝自动编辑包含 merge 提交的历史段，避免破坏分支拓扑。

还原提交时如果出现冲突，工作区会标出冲突文件，并显示“继续还原 / 中止还原”。解决冲突并暂存后继续；不想保留这次还原则中止。

挑选提交时如果出现冲突，工作区会标出冲突文件，并显示“继续挑选 / 跳过挑选 / 中止挑选”。解决冲突并暂存后继续；不想保留当前提交可跳过，不想继续这次挑选可中止。

## Git 储藏

储藏相当于把当前未提交的改动临时收进一个 Git stash。储藏后，文件会从工作区列表里消失，这是正常行为；这些改动没有丢，会出现在右侧“储藏”页，可以查看 Diff、应用、弹出或删除。
