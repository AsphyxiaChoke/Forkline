# Forkline

中文 Web 端 Git 可视化管理工具。无需安装 npm 依赖，使用 Node.js 内置 HTTP 服务启动。

## 功能

- GitKraken 风格提交图谱，全部分支模式下以 `main` / `master` 为主线；顶部搜索可按提交信息、作者、分支/标签或 SHA 分词过滤，并显示命中数量和高亮。
- 支持命令面板：可搜索并执行常用页面跳转、同步、工作区、分支、Tag、克隆和初始化操作，危险动作保持红色标识并复用原确认流程。
- 顶部支持最近仓库快速打开，会在当前浏览器记住最近打开过的真实 Git 仓库，便于在多个项目之间切换。
- 支持从远端 URL 或本地裸仓库克隆到指定本机目录，克隆完成后可直接打开新仓库。
- 支持把本机普通文件夹初始化为 Git 仓库，初始化后可直接打开；刚初始化且还没有提交的空仓库会显示稳定空状态。
- 本地分支、远端分支、工作区状态和未暂存更改查看；左侧分支列表可按分支名或 upstream 筛选，本地分支会显示 upstream、领先/落后数量和上游丢失状态。
- 右侧“分支”页支持分支整理：汇总已合并、上游丢失和长期未动分支，标出当前分支、主干分支和 worktree 占用原因，并可安全删除单个分支或批量删除已合并分支。
- 右侧“同步”页会显示当前分支 upstream、同步建议、待拉取提交、待推送提交和远端仓库配置，并提供抓取 / 拉取 / 变基拉取 / 推送 / 安全强推 / 添加远端 / 诊断连接 / 修改 URL / 删除远端入口；远端诊断结果会留在同步页，显示 URL、命令、中文原因、问题分类、排查步骤和可复制诊断命令。
- 同步页内置认证助手：按远端 URL 判断 SSH / HTTPS / 本地路径，列出本机 `~/.ssh` 中可见的 SSH key 文件名，检测 `ssh-agent` 和 Git Credential Manager，并提供可复制诊断命令。
- 同步页可为当前分支生成 Pull Request / Merge Request 创建链接，支持 GitHub、GitLab、Bitbucket 和常见 Gitea / Forgejo 网页远端；入口也在命令面板和当前分支右键菜单中，可直接打开或复制链接。
- 同步页的待拉取 / 待推送提交可直接点击预览文件列表和 Diff，并可最大化对照查看。
- 提交详情和提交右键菜单支持在 GitHub / GitLab / Bitbucket / Gitea 等网页远端中打开当前提交。
- 右侧“日志”页会显示进行中的 Git 操作，并记录最近 Git 操作的成功/失败、耗时和 Git 输出摘要，便于排查哪个操作正在卡住或刚刚失败。
- 支持当前分支 upstream 管理：同步页可从已抓取的远端分支中设置 upstream 或取消 upstream；远端分支右键菜单也可直接设为当前分支 upstream。
- 支持普通推送保护：当前分支落后 upstream 或与 upstream 分叉时，Forkline 会阻止普通 `git push`，同步页显示中文保护原因，并引导先“变基拉取”或在确认改写历史时使用安全强推。
- 文件路径树状归纳，支持同层级聚合与滚动查看；工作区文件可按路径、文件名、状态和暂存状态筛选，未跟踪文件可从右键菜单直接加入 `.gitignore`。
- 工作区 Diff、历史提交 Diff、小窗口预览和最大化对照；工作区 Diff 面板可在“未暂存 / 已暂存”之间切换，可直接暂存、取消暂存、丢弃当前文件，也支持对已跟踪文件按改动块暂存、取消暂存或丢弃未暂存块；未跟踪文本文件会按虚拟块展示，可只暂存选中的块。
- 支持文件历史：工作区文件右键或提交文件面板可查看单个文件的 `git log --follow` 历史，并跳转到相关提交或该文件的历史改动。
- 支持逐行追踪：工作区文件右键或提交文件面板可查看 `git blame` 结果，定位每一行对应的提交、作者、时间和摘要。
- 可拖拽左侧栏、右侧栏、底部变更面板、文件列表宽度和提交框高度。
- 支持抓取、快进拉取、变基拉取、智能推送和安全强推，并在完成后显示中文同步摘要：当前 upstream、领先/落后变化、远端新增/更新/删除引用；没有 upstream 的当前分支会自动使用 `git push -u` 设置跟踪关系。
- 支持暂存全部、单文件与 Ctrl / Shift 多选暂存 / 取消暂存、丢弃未暂存或已暂存改动、创建提交和追加提交。
- 支持创建储藏、储藏所选文件、查看储藏 Diff、应用 / 弹出 / 删除储藏，以及从储藏创建并切换到新分支。
- 支持本地分支切换、远端分支签出为本地跟踪分支、远端分支右键删除、保留 merge commit 的分支合并、新建/重命名/删除分支。
- 支持比较任意两个引用：可从分支右键菜单比较当前分支与本地/远端分支，也可在右侧“比较”页直接输入本地分支、远端分支、Tag 或提交 SHA；比较结果会显示两边独有提交、文件变化和可最大化 Diff。
- 支持创建 Tag、标签列表、复制/查看 Tag、删除本地 Tag、推送 Tag、删除远端 Tag，相关动作也集成在 Tag 右键菜单中。
- 支持合并冲突工作流：冲突时工作区会显示继续 / 中止合并；冲突文件可一键使用当前版本或对方版本解决并暂存。
- 支持从分支右键菜单把当前分支变基到目标分支；变基冲突时工作区会显示继续 / 跳过 / 中止变基。
- 支持在提交右键菜单和提交详情中挑选提交到当前分支；merge 提交会先选择主线，冲突时工作区会显示继续 / 跳过 / 中止挑选。
- 支持在提交右键菜单和提交详情中还原提交；merge 提交会先选择主线；支持软 / 混合 / 硬重置到指定提交，并在按钮旁标注对应 Git 指令。
- 支持修改提交信息：最新提交使用 amend，历史提交使用 rebase reword；提交详情和右键菜单支持把非 merge 提交压缩进父提交、修补进父提交或从历史中丢弃，执行前会先展开历史编辑计划并显示影响范围、阻塞原因和恢复点提示；也支持把多条 squash / fixup / drop / reword 动作加入历史编辑队列，队列项可直接改动作、填写新的提交摘要/正文并调整显示顺序，统一预检后一次执行。
- 支持自动恢复点：追加提交、变基拉取、分支变基、历史编辑和重置前会自动创建隐藏恢复引用，右侧“恢复点”页可搜索、按分支/动作筛选、恢复、删除、批量清理，或按“保留最近 N 天 / 每个分支保留 N 个”策略清理；保留策略会记住到当前浏览器，清理前会展开候选恢复点。

## 启动

```bat
start.cmd
```

打开：

```text
http://127.0.0.1:5177
```

默认显示示例数据。顶部输入本机 Git 仓库路径后点击 `打开`，即可读取真实仓库。

成功打开真实仓库后，Forkline 会把仓库路径保存到当前浏览器的“最近仓库”下拉框中。这个记录只存在浏览器本地存储里，清除最近仓库不会删除任何本地 Git 仓库。

顶部“克隆”会执行 `git clone <来源> <保存到>`。目标必须是本机绝对路径；如果目标文件夹已经存在且不是空文件夹，Forkline 会拒绝覆盖。

顶部“初始化”会执行 `git init <文件夹>`。文件夹必须是本机绝对路径；如果目标不存在，Forkline 会创建这个文件夹；如果目标已经是 Git 仓库，会中文拒绝重复初始化。普通非 Git 文件夹可以直接初始化，已有文件会保留在工作区中等待第一次提交。

本地静态资源响应使用 `Cache-Control: no-store`，开发时刷新页面会直接拿到最新的 `app.js` / `styles.css`。

## 注意

真实仓库模式下的写入操作会先弹窗确认，再由本地 Node 服务执行 Git 命令。修改历史提交信息和变基都会重写相关 SHA，工作区不干净时会被阻止。

抓取、拉取和推送完成后，Forkline 会显示同步摘要。摘要会说明当前分支对应的 upstream、本地领先/落后数量如何变化，以及这次远端引用新增、更新或删除了什么。

推送当前分支时，如果已经设置 upstream，Forkline 会执行普通 `git push`；如果没有 upstream，会选择 `origin` 或第一个可用远端执行 `git push -u <远端> <当前分支>`。删除远端分支会执行 `git push <远端> --delete <分支>`，只删除远端仓库中的分支，不会删除本地分支。

同步页里的远端仓库管理对应 `git remote add`、`git remote set-url`、`git remote remove`、`git fetch <远端> --prune` 和 `git ls-remote --heads <远端>`。删除远端只会移除当前本地仓库的远端配置，不会删除 GitHub 或服务器上的仓库。“诊断连接”只读取远端分支列表，用来确认远端 URL、网络和认证是否可用，不会写入本地引用；诊断结果会在同步页保留，失败时会按 SSH key、HTTPS token、仓库权限、DNS/代理、证书或 URL/本地路径分类，给出中文排查步骤和可复制命令。

同步页的认证助手只读取认证环境摘要，不读取或显示私钥内容。SSH key 清单只展示文件名、是否有公钥/私钥配对和更新时间；`ssh-agent` 检测执行 `ssh-add -l`，HTTPS 凭据检测会尝试 `git credential-manager version` / `git credential-manager-core --version`。这些检查用于判断环境是否具备认证条件，真正的远端权限仍以“诊断连接”的 `git ls-remote --heads <远端>` 结果为准。

同步页的 PR/MR 链接只根据当前本地分支和已识别的网页远端生成，不会创建远端分支或提交网页表单。Forkline 会优先使用当前分支的 upstream 作为目标分支；如果 upstream 和当前分支同名，则回退到 `main` / `master` / `develop` / `dev` / `trunk` 等主干分支。本地路径远端不会生成 PR/MR 链接。

设置 upstream 会执行 `git branch --set-upstream-to=<远端分支> <当前分支>`；取消 upstream 会执行 `git branch --unset-upstream <当前分支>`。Forkline 只允许给当前本地分支设置已抓取到的远端分支，避免误把不存在的引用写进配置。

安全强推会执行 `git push --force-with-lease <远端> HEAD:<分支>`，用于 rebase、squash、amend 等改写历史之后更新远端分支。它不会使用裸 `--force`；如果远端分支在你上次抓取后又变化，Git 会拒绝这次强推。

普通推送前，Forkline 会检查当前分支与 upstream 的领先/落后关系。如果本地落后远端，或本地领先同时落后形成分叉，普通推送会被保护拦截，避免非快进推送失败或误覆盖协作历史。同步页会显示待拉取/待推送提交，方便先判断应该拉取、变基拉取还是安全强推。

“拉取”执行 `git pull --ff-only`，只允许快进，避免偷偷生成 merge commit。“变基拉取”执行 `git pull --rebase`，会先拉取 upstream，再把本地未推送提交重放到远端提交之后；这些本地提交的 SHA 可能改变。遇到冲突时，工作区会进入变基冲突流程，并显示继续 / 跳过 / 中止变基。

推送 Tag 会执行 `git push <远端> refs/tags/<tag>:refs/tags/<tag>`；删除远端 Tag 会执行 `git push <远端> :refs/tags/<tag>`，不会删除本地 Tag。本地删除 Tag 使用 `git tag -d <tag>`，不会影响远端 Tag。

对 merge 提交执行挑选或还原时，Forkline 会要求选择主线，对应 Git 的 `-m <父提交编号>`。通常父提交 1 是执行合并时所在的分支方向，父提交 2 是被合并进来的分支方向。

如果 Git 提示 `index.lock`，Forkline 会在中文提示里显示刚才执行的操作、锁文件路径/时间，以及能检测到的 Git 进程。`index.lock` 本身不记录具体命令；确认没有 Git 或编辑器正在操作仓库后，才能删除这个锁文件重试。

合并分支时如果出现冲突，工作区会标出冲突文件，并显示“继续合并 / 中止合并”。冲突文件右键菜单和底部 Diff 面板可执行“使用当前版本 / 使用对方版本”，对应 `git checkout --ours -- <文件>` / `git checkout --theirs -- <文件>` 后自动 `git add <文件>`；解决并暂存后继续，不想保留这次合并则中止。变基、挑选、还原冲突也复用同一套文件级解决入口。

分支右键菜单里的“与当前分支比较”会读取 `git rev-list --left-right --count <当前分支>...<目标分支>`，并用 `git diff <当前分支>...<目标分支>` 展示目标分支相对共同祖先带来的文件变化。这个比较支持本地分支和已抓取到的远端分支。

右侧“分支”页的整理列表会读取本地分支最后提交、upstream 状态和 `git branch --merged HEAD`。单个删除和“删除已合并”都执行安全删除 `git branch -d <分支>`；如果分支还没有完全合并，Git 会阻止删除。Forkline 默认保护当前分支和 `main` / `master` / `develop` / `dev` / `trunk` 这类主干分支，也会禁用被其他 worktree 占用的分支。

文件历史会执行 `git log --follow --find-renames <引用> -- <文件>`，用于追踪单个文件的新增、修改、删除和重命名记录。Forkline 只接受仓库内相对路径，避免读取仓库外文件。

工作区 Diff 的“未暂存 / 已暂存”切换会分别读取 `git diff` 和 `git diff --cached`。当同一个文件既有已暂存改动又有未暂存改动时，可以切到对应视图查看和操作。“暂存此块 / 取消暂存此块 / 丢弃此块”会重新读取当前文件的真实 Git diff，再用 `git apply` 只应用选中的 hunk。未跟踪文本文件会按 40 行左右生成虚拟块，可以只把选中的块暂存到索引；未跟踪文件的块级丢弃不开放，需要在编辑器里修改文件内容。

未跟踪文件右键菜单里的“加入 .gitignore / 忽略所在目录”只会向仓库根目录的 `.gitignore` 追加规则，例如 `/tmp.log` 或 `/dist/`，不会删除本地文件，也不会自动暂存 `.gitignore`。已跟踪文件不会因为加入 `.gitignore` 自动停止跟踪，需要先从 Git 索引中移除后才能忽略。

逐行追踪会执行 `git blame --line-porcelain <引用> -- <文件>`，右侧“逐行”页会显示行号、提交短 SHA、作者和代码内容。大文件会限制返回前 600 行，避免浏览器卡顿；Forkline 同样只接受仓库内相对路径。

变基当前分支到其他分支时，Forkline 会执行 `git rebase <目标分支>`。这会重写当前分支中被重放提交的 SHA；如果这些提交已经推给别人协作，应先确认团队是否允许。变基冲突时，工作区会标出冲突文件，并显示“继续变基 / 跳过变基 / 中止变基”。解决冲突并暂存后继续；不想保留当前冲突提交可跳过；想回到变基前状态则中止。

历史编辑里的“压缩进父提交 / 修补进父提交 / 丢弃此提交”对应 `git rebase -i` 的 `squash`、`fixup`、`drop`。点击后 Forkline 会先在提交详情中展开计划，显示当前分支、目标提交、父提交、重放范围、影响提交数量和阻塞原因；确认执行后才会重写当前分支历史 SHA。Forkline 会要求工作区干净，并暂时拒绝自动编辑包含 merge 提交的历史段，避免破坏分支拓扑。

历史编辑队列可以从提交详情或提交右键菜单把多条提交加入同一个队列。队列项可以直接改为 squash、fixup、drop 或 reword，也可以上移/下移整理显示顺序；reword 队列项会显示新的提交摘要和正文输入框。Forkline 会统一预检队列组合，并在预览区显示按当前分支历史生成的实际执行顺序。预检会拦截 merge 提交、脏工作区、重复提交、reword 与 squash/fixup 混用、以及“前一条提交会被丢弃但后一条还要 squash/fixup”的危险组合；执行队列前同样会创建恢复点。

执行追加提交、变基拉取、分支变基、修改提交信息、历史编辑或 reset 前，Forkline 会在 `refs/forkline/recovery/...` 下创建隐藏恢复点，不会污染普通分支列表。右侧“恢复点”页可以搜索、按分支/动作筛选、批量清理，也可以按“保留最近 N 天 / 每个分支保留 N 个”策略清理旧恢复点，或执行 `git reset --hard <恢复点>` 回到对应提交；恢复前 Forkline 会再创建一个新的恢复点。恢复点保留策略会保存到当前浏览器，策略卡会在清理前展示将删除的候选恢复点。恢复点只保存提交位置，不保存未提交的工作区改动。

右侧“日志”页显示当前正在执行的 Forkline 操作，以及本地服务启动后的最近 Git 操作。它会记录操作名称、动作类型、成功/失败、耗时和 Git 输出摘要；刷新页面不会清空日志，重启本地服务后日志会重新开始。

还原提交时如果出现冲突，工作区会标出冲突文件，并显示“继续还原 / 中止还原”。解决冲突并暂存后继续；不想保留这次还原则中止。

挑选提交时如果出现冲突，工作区会标出冲突文件，并显示“继续挑选 / 跳过挑选 / 中止挑选”。解决冲突并暂存后继续；不想保留当前提交可跳过，不想继续这次挑选可中止。

## Git 储藏

储藏相当于把当前未提交的改动临时收进一个 Git stash。储藏后，文件会从工作区列表里消失，这是正常行为；这些改动没有丢，会出现在右侧“储藏”页，可以查看 Diff、应用、弹出、删除，或执行 `git stash branch <分支> <储藏>` 从这条储藏创建新分支。成功从储藏创建分支后，Git 会切到新分支并移除对应储藏。
